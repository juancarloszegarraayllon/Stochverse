## 1. Cache record schema

Source: `python3 kalshi_probe/probe_cache_schema.py` ŌåÆ
`/api/_debug/cache_schema`. See full field list there.

**Stable, universally-populated fields** (100% across all sports):

| Field | Type | Meaning |
|---|---|---|
| `event_ticker` | str | Kalshi event ticker (the join key) |
| `series_ticker` | str | Kalshi series ŌĆö e.g. `KXEPLGAME` |
| `title` | str | Human-readable, e.g. `"Bayern Munich vs PSG"` |
| `category` | str | Kalshi top-level category (`Sports`, `World`, etc.) |
| `outcomes` | list | Per-outcome `{label, prob, yes, no, ticker}` |
| `_sport` | str | Set by our cache builder, e.g. `Soccer` |
| `_close_dt` / `_exp_dt` | str (ISO) | Market close / expiration |

**Key fact**: `_live_state` is **not** stored on cache records.
`/api/events` computes it per request via `match_game` +
`_enrich_soccer_aggregate`. Anything that needs aggregate / series /
display_clock fields must run the same enrichment chain.

## 2. Series tickers per sport

Source: `probe_series.py` ŌåÆ `/api/_debug/series_tickers`.

Series tickers split by suffix into per-fixture markets and
season-long outrights. Suffixes observed in the wild:

```
GAME, MATCH       ŌĆö per-fixture h2h headline
1H                ŌĆö first-half winner (soccer only)
F5                ŌĆö first 5 innings (baseball only)
TOTAL             ŌĆö totals / over-under
SPREAD            ŌĆö spread / handicap
BTTS              ŌĆö both teams to score (soccer only)
1Q, 2Q, 3Q, 4Q    ŌĆö quarter markets (basketball)
2H                ŌĆö second-half markets (basketball)
OVERTIME          ŌĆö binary OT market
ADVANCE           ŌĆö two-leg tie advancement (soccer cups)
CORNERS           ŌĆö total corners (soccer)
TCORNERS          ŌĆö team corners (soccer)
HALFTIME          ŌĆö halftime score
""  (no suffix)   ŌĆö outright / future / season-long / props
```

**Suffix is a deterministic predictor of outcome shape** (see ┬¦4).

## 3. Title shapes per sport

Source: `probe_titles.py` ŌåÆ `/api/_debug/title_shapes`.

**Universal title shapes**:

| Shape | Meaning | Used by |
|---|---|---|
| `"X vs Y"` | h2h matchup, left=home, right=away | GAME headline (Soccer / NFL / UFL / MLB) |
| `"X at Y"` | h2h matchup, left=away, right=home | TOTAL/SPREAD/BTTS/RFI/F5/etc. (NBA, MLB, soccer sub-markets) |
| `"X vs Y: <market>"` | sub-market on h2h fixture | 1H, ADVANCE, CORNERS, TCORNERS, OVERTIME |
| `"Game N: X at Y"` | playoff series leg | NBA / NHL playoff games |
| `"Series <Type>: A (s) vs B (s)"` | NBA / NHL series-level market | KXNBASERIES, KXNHLSERIES |
| `"<Team> Total Wins"` | per-team season win futures | KXMLBWINS-{TEAM}, KXNFLWINS via market_type |
| free-form question | outrights / futures / yes-no | everything else (no suffix) |

**Title preposition encodes orientation** (universal):

| Preposition | Meaning |
|---|---|
| `vs` | left = home, right = away |
| `at` | left = **away** (visiting), right = home |

US-sports convention: `at` always means visiting team plays at home team's venue.

### Tennis

| Suffix | market_type | Outcomes | Labels |
|---|---|---|---|
| `MATCH` | ŌĆö | 2 | player full names (NO TIE ŌĆö tennis can't tie) |
| `SETWINNER` (KXATPSETWINNER, KXWTASETWINNER) | `Set 1 Winner` / `Set 2 Winner` | 2 | player full names |
| `GRANDSLAM` (per-player, KXGRANDSLAM) | `Grand Slam wins in 2026` | 3 | `1+`, `2+`, `3+`, `All 4 Grand Slam wins` |
| Outright tour winners (KXFOMEN, KXFOWOMEN, KXATPGRANDSLAM, KXWTAGRANDSLAM) | ŌĆö | 14ŌĆō25 | player full names |

**Tennis sub-tiers** (all use the same `MATCH` shape, different bases for tour level):
- `KXATP` ŌĆö men's tour (12 records)
- `KXATPCHALLENGER` ŌĆö men's challenger tier (19 records)
- `KXWTA` ŌĆö women's tour (34 records)
- `KXWTACHALLENGER` ŌĆö women's challenger (18 records)
- `KXITF` ŌĆö men's ITF lower tier (60 records)
- `KXITFW` ŌĆö women's ITF lower tier (133 records ŌĆö biggest tennis bucket)

**Title-shape quirk**: tennis uses **surname-only** in `MATCH` titles (`"Hooper vs Ponchet"`) but **full names** in `SETWINNER` titles (`"Taylor Townsend vs Victoria Jimenez Kasintseva: Set 1 Winner"`). Same player pair, two different title shapes depending on suffix. **The renderer should always read player labels from `outcomes[].label`** (which always has the full name), not from titles.

### Universal: outrights / futures (no suffix)

Outcome count is variable (1 to 70+), labels are sport-specific:
- 1 outcome: binary YES/NO speculation (e.g., `Will Lamine Yamal leave Barcelona`)
- 2ŌĆō48 team_like: champion / division / conference / playoff / draft / squad
- 10ŌĆō72 player_like: MVP / ROY / Cy Young / Calder / Goal Leader / Top RB / etc.
- 16ŌĆō225 pair labels: `Finals Matchup` (e.g., `PSG vs Atletico`)

### Universal facts
- **Field coverage = 100%** for `prob` / `yes` / `no` / `ticker` across every sport, every bucket. Kalshi reliably ships prices everywhere; missing prices in our UI are bugs in our code, not in their data.
- **Outcome label IS the team name** in `GAME` markets ŌĆö no parsing needed.
- **Outcome label starts with team name** in `SPREAD` markets ŌĆö extractable via prefix match.

## 5. Ticker grammar (per sport)

Source: `probe_tickers.py` ŌåÆ `/api/_debug/ticker_grammar`.

### G1 ŌĆö date + teams (no time)

```
{base}{suffix}-{YYMMDD}{HOME_ABBR}{AWAY_ABBR}
```

Used by:
- **All Soccer** leagues (KXEPL, KXLALIGA, KXSERIEA, KXBUNDESLIGA, KXLIGUE1, KXMLS, KXJLEAGUE, KXKLEAGUE, KXSAUDIPL, KXCHNSL, KXBRASILEIRO, KXUCL, KXUEL, KXUECL, KXCONMEBOLLIB, KXCONMEBOLSUD, KXUAEPL, KXEGYPL, KXARGPREMDIV, KXNWSL, KXEWSL, KXBOLPDIV, KXURYPD, KXCONCACAFCCUP, KXWC)
- **NBA** (KXNBA + all sub-markets including player props)
- **NHL** (KXNHL + sub-markets)
- **UFL**

Examples:
- `KXEPLGAME-26MAY19CFCTOT` ŌåÆ date 2026-05-19, home CFC, away TOT
- `KXNBAGAME-26MAY11OKCLAL` ŌåÆ home OKC, away LAL
- `KXNHLOVERTIME-26MAY10VGKANA` ŌåÆ home VGK, away ANA
- `KXUFLGAME-26MAY10ORLHOU` ŌåÆ home ORL, away HOU

**Date format**: `YY` + 3-letter UPPERCASE month + `DD`. Locale-independent, UTC-anchored.

**Team block**: variable length 4ŌĆō8 chars total. Parsed by **walking from the right** to the longest known team-abbr suffix, then home_abbr is what's left. Real-world cases:
- `OKCLAL` (3+3) ŌĆö Oklahoma City + LA Lakers
- `LAFCHOU` (4+3) ŌĆö LAFC + Houston
- `M05UNI` (3+3 with digit) ŌĆö Mainz 05 + Union Berlin
- `PITAZ` (3+2) ŌĆö Pittsburgh + Arizona

**Team abbr regex must allow digits**: `[A-Z0-9]+` (not `[A-Z]+`). Five MLS / Bundesliga tickers in our sample fail a strict-alpha regex.

### G7 ŌĆö date + time + teams

```
{base}{suffix}-{YYMMDD}{HHMM}{HOME_ABBR}{AWAY_ABBR}
```

Used by:
- **All MLB** per-fixture markets (because doubleheaders)
- **KBO**, **NPB** (baseball)
- **AHL**, **SHL**, **KHL**, **DEL**, **Liiga** (international hockey)
- **EuroLeague**, **BBL**, **CBA**, **ISL**, **ABA**, **ARGLNB**, **FIBACHAMPLEAGUE**, **LNBELITE**, **BSL**, **JBLEAGUE**, **NZNBL**, **GBL**, **KBL**, **ACB** (international basketball)

Examples:
- `KXMLBGAME-26MAY071540PITAZ` ŌåÆ date 2026-05-07, time 15:40, home PIT, away AZ
- `KXEUROLEAGUEGAME-26MAY061415VALPAN` ŌåÆ time 14:15
- `KXAHLGAME-26MAY061900MANGRA` ŌåÆ time 19:00

**Time is part of the fixture identity** ŌĆö same team-pair on same date can mean two different games (doubleheader).

### G3 ŌĆö year (4-digit)
```
{base}-{YYYY}     e.g. KXPFAPOY-2526, KXPLAYTOGETHERJBJT-2027
```

### G4 ŌĆö short year (2-digit)
```
{base}-{YY}       e.g. KXUCL-26, KXNBAMVP-26, KXSB-27
```

Used by virtually every outright / season-long / future market across all sports.

### G5 ŌĆö handle
```
{base}-{HANDLE}   e.g. KX1STHOMEGAME-BUF26, KXNEXTNBACOACH-POR26
```

### G_SERIES ŌĆö NBA / NHL playoff series (round-scoped)
```
{base}-{YY}{HOME_ABBR}{AWAY_ABBR}R{N}
```

Examples: `KXNBASERIES-26LALOKCR2`, `KXNHLSERIESSCORE-26MTLBUFR2`

`R{N}` is the playoff round number. No date ŌĆö the market spans the full series.

### G_SET ŌĆö set-winner sub-markets (Tennis)

```
{base}-{YYMMDD}{HOME_ABBR}{AWAY_ABBR}-{N}
```

`N` is the set number (1 or 2). Used by `KXATPSETWINNER`, `KXWTASETWINNER`.

Examples: `KXATPSETWINNER-26MAY05HIJBAS-1`, `KXWTASETWINNER-26MAY05TOWJIM-2`.

The base ticker (without `-{N}`) matches the corresponding `MATCH` ticker, so a set-winner record can be deterministically associated with its parent match.

### G_DATE_ONLY ŌĆö date-keyed outright (Tennis)

```
{base}-{YYMMDD}
```

Used for "ranked / state on date X" type futures. Examples: `KXATP1RANK-26DEC31`, `KXATP1RANK-26JUN30`.

### Tennis player abbreviations

Tennis uses **first 3 chars of surname** as the player abbreviation, concatenated home+away ŌåÆ 6 chars total. Examples:
- `HIJBAS` ŌåÆ Hijikata + Basilashvili
- `TOWJIM` ŌåÆ Townsend + Jimenez Kasintseva
- `MILPAV` ŌåÆ Milovanova + Pavlova

**Edge case**: compound surnames can produce 7-char blocks. `KXATPCHALLENGERMATCH-26MAY05DIABER` ŌåÆ Diaz Acosta (4-char `DIA`) + Bernet (`BER`). Rare (1 in our sample).

**Edge case 2**: rematch / same-day-second-bracket. `KXITFWMATCH-26MAY04CHOCHO2` ŌĆö same `CHOCHO` abbrs with trailing `2`. Rare (~0.75% of ITFW matches). Renderer should keep the `2` as part of fixture identity to avoid collisions.

### Sport-specific outright ticker variations

Some bases use unique encodings that don't fit G1ŌĆōG7. They appear only in non-h2h markets, so they don't affect FL pairing ŌĆö list provided for completeness:

- `KXMLBWINS-{TEAM_ABBR}-{YY}` ŌĆö e.g. `KXMLBWINS-WSH-26` (per-team season wins)
- `KXNEXTTEAMNFL-{YY}{PLAYER}` ŌĆö e.g. `KXNEXTTEAMNFL-27JALLEN`
- `KXSTARTINGQBWEEK1-W1-{YYMMDD}-{TEAM}` ŌĆö `KXSTARTINGQBWEEK1-W1-26SEP15-PIT`
- `KXMLBSTAT-{YY}{CODE}` ŌĆö `KXMLBSTAT-26W20`, `KXMLBSTAT-26HR500`, `KXMLBSTAT-26PG`
- `KXMLBSEASONHR-{YY}C{N}` ŌĆö `KXMLBSEASONHR-26C30`
- `KXNBADRAFTPICK-{YY}-{N}` ŌĆö `KXNBADRAFTPICK-26-3`
- `KXWCGROUP{QUAL,WIN}-{YY}{LETTER}` ŌĆö `KXWCGROUPQUAL-26L`
- `KXWCROUND-{YY}{ROUND}` ŌĆö `KXWCROUND-26SEMI`
- `KXWCSQUAD-{YY}{COUNTRY3}` ŌĆö `KXWCSQUAD-26BRA`

## 7. Merge contract: Kalshi Ōåö FlashLive

**The deterministic rule that /sports v2 implements.**

### 7.1 Kalshi-side identity

For each Kalshi cache record `r`:

```
sport         = r._sport                                      (set at cache build)
series_base   = strip_known_suffix(r.series_ticker)
suffix        = the stripped suffix (or "" for no suffix)
fixture_kind  = classify(series_base, suffix)
                  Ōłł { per_fixture, series, per_team, outright }
```

**If `fixture_kind == per_fixture`**:
```
parsed = parse_ticker(r.event_ticker, sport)   # G1 or G7 per ┬¦5

kalshi_identity = (
    sport,
    parsed.date,
    parsed.time,                  # None for G1, HHMM for G7
    frozenset({parsed.home_abbr, parsed.away_abbr}),  # order-agnostic
)
```

**If `fixture_kind == series`** (NBA / NHL playoff series):
```
parsed = parse_series_ticker(r.event_ticker)

series_identity = (
    sport,
    series_base,
    parsed.year,
    frozenset({parsed.home_abbr, parsed.away_abbr}),
    parsed.round,
)
```

**If `fixture_kind == per_team`** (KXMLBWINS, KXNFLWINS, etc.):
```
team_identity = (sport, series_base, year, team_abbr)
```

**If `fixture_kind == outright`** (champion, MVP, ROY, futures, ŌĆ”):
```
outright_identity = (sport, series_base, year)
```

### 7.2 FL-side identity (per FL event `e`)

```
fl_identity = (
    sport,                                       # FL sport_id ŌåÆ name
    e.START_TIME ŌåÆ date (UTC),
    e.START_TIME ŌåÆ time HHMM (UTC),
    frozenset({e.SHORTNAME_HOME, e.SHORTNAME_AWAY}),
)
```

### 7.3 The pairing rule

```
match(k, e) iff
    k.sport == e.sport
  AND k.kalshi_identity.date == e.fl_identity.date    (┬▒1 day fuzz for tz drift)
  AND k.kalshi_identity.team_set == e.fl_identity.team_set
  AND IF k.kalshi_identity.time is not None:
        |k.time - e.fl_identity.time| <= 30 minutes   (kickoff drift tolerance)
```

This is a **deterministic equality** check on parsed identity tuples. Note what's *not* in the rule:
- ŌØī no `match_game` fuzzy team-name search
- ŌØī no `_kalshi_title_corroborates_fl_game` token gymnastics
- ŌØī no second-pass attach pass
- ŌØī no team-alias dictionary
- ŌØī no nickname expansion

The complexity in the current code exists because we were inferring identity from titles + team names. With identity computed from tickers + SHORTNAMEs, all of that goes away.

### 7.4 Renderer rule

Once paired (or for stand-alone Kalshi-only events), the renderer
builds outcomes per the `(sport, suffix)` table in ┬¦4. No
shape-guessing from titles.

### 7.5 League bucket rule

Bucket assignment is keyed by `series_base` ŌåÆ FL `tournament_stage_id`,
learned from observed pairings (with cross-request persistence).
No SOCCER_COMP-style hand-curated map needed for bucketing
(it's still useful for **labels** when no FL fixture is paired,
but not for routing).

### 7.6 What this fixes

| Past bug | Why it happened | How the contract prevents it |
|---|---|---|
| Same fixture in two leagues | Synthesized bucket name ŌēĀ FL bucket name | series_base routing keyed by observed pairings |
| Missing away team in card | Outcome-label parsing assumed shape | Per-(sport,suffix) outcome table |
| Wrong-leg pairing for cup ties | match_game returned earlier leg | Date is in the ticker ŌĆö wrong leg can't match |
| Atl. Madrid vs Atletico no match | Token corroboration | Tickers use SHORTNAME `ATM` directly |
| Bayern Munich vs PSG no match | match_game phrase miss | SHORTNAMEs do the work, no phrase matching |
| Duplicates from suffix splits | matchup-key inconsistencies | Identity is parsed, not derived from title |

## 8. What this doc replaces

When /sports v2 is built, the following code can be deleted in
favor of the contract above:

- `_kalshi_title_corroborates_fl_game()` ŌĆö replaced by ticker parse + SHORTNAME equality
- `_collect_unpaired_h2h_for_sport()`'s `match_game` + corroboration loop ŌĆö replaced by deterministic identity
- `_build_kalshi_index_for_sport()`'s sanity-check rejections ŌĆö replaced by direct lookup by parsed identity
- `_matchup_key()` and the matchup-dedup logic ŌĆö replaced by identity tuple
- The "second-pass attach" pass added in `a985a95` ŌĆö no longer needed
- The "series-routing pass" added in `8232b90` ŌĆö kept conceptually but driven by parsed series_base, not synthesized names
- `TEAM_ALIASES` ŌĆö kept only for **rendering** (e.g., showing "Manchester United" instead of "Man United") not for **pairing**

## Regenerating this document

Each section corresponds to a probe; rerunning the probe gives the
fresh data. To pull current state for any section, hit the matching
endpoint:

```
/api/_debug/cache_schema?sport=<sport>
/api/_debug/title_shapes?sport=<sport>
/api/_debug/series_tickers?sport=<sport>
/api/_debug/outcome_shapes?sport=<sport>
/api/_debug/ticker_grammar?sport=<sport>
```

When Kalshi adds a new series or sport, the probes flag it (any
bucket appearing in the output that's not in this doc is a new
shape that needs deciding before /sports renders it).
